REST
` REpresentational State Transfer의 줄임말
` 서버의 자원 정의 및 자원에 대한 주소 지정 방법
`   ` 자원 : 서버가 행할 수 있는 모든 것
` HTTP 요청 메소드 사용 : 클라이언트에 상관 없이 같은 방식으로 서버와 소통 가능 ( = 서버와 클라이언트의 분리 )
`   ` Get : 서버 자원을 가져올 때 사용 ; 요청 본문에 데이터 X, 서버로 데이터 전송 시 쿼리스트링 사용 / 브라우저에서 캐싱(기억) 가능
`   ` Post : 서버에 자원을 새로 등록 시 사용 ; 요청 본문에 등록할 데이터 포함해 전송
`   ` Put : 서버 자원을 요청에 든 자원으로 치환 시 사용 ; 요청 본문에 치환할 데이터 포함해 전송
`   ` Patch : 서버 자원의 일부만 수정 시 사용 ; 요청 본문에 수정할 데이터 포함해 전송
`   ` Delete : 서버 자원 삭제 시 사용 ; 요청 본문에 데이터 X
`   ` Options : 요청 전 통신 옵션 설명을 위해 사용

` 주소 하나 당 요청 메소드가 다수일 수 있음

` RESTful : REST를 따르는 서버
` 코드 작성 전 각 주소의 구조를 미리 표 등으로 그려놓을 것
